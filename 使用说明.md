# 京东搜索列表抓取脚本说明

## 文件说明

| 文件 | 作用 |
|------|------|
| `jd_search_crawl.py` | 单关键词：搜索、监听、翻页、滚动收包、写 CSV。支持 `--max-rows` 限制行数。 |
| `jd_category_jobs.py` | 九类「品类+自营」与输出文件名的对照表（可改可扩）。 |
| `jd_multi_category_crawl.py` | 按上表批量跑九类，每类一个 CSV，默认每类约 500 条。 |

逻辑与 `test.ipynb` 中当前方案一致：`listen.wait` 必须使用关键字参数 `timeout=...`（第一个位置参数是「包个数」`count`，不是秒数）。

## 环境要求

1. **Python 3.10+**（推荐 3.11）。
2. 已安装 **DrissionPage**，并已配置 **Chrome 或 Edge** 路径（项目内可改 `config_address.py` 后执行 `python config_address.py` 保存；**勿配置 Firefox**，`ChromiumPage` 仅支持 Chromium 内核）。命令行也可用 `--chrome-path` 或环境变量 `JD_CHROME_PATH`（见 `jd_chromium.py`）。
3. 能正常用浏览器手动打开 [京东搜索](https://search.jd.com)。

## 安装依赖

```bash
pip install DrissionPage
```

## 用法示例

在项目目录下执行（请把路径换成你的实际目录）：

```bash
cd d:\code\claude_code\self_support_crawl

# 默认关键词 cpu自营，输出 jd_search_result.csv，最多 100 页（与页面总页数取小）
python jd_search_crawl.py

# 指定关键词与输出文件
python jd_search_crawl.py -k "手机" -o phones.csv

# 只抓前 5 页（仍会与「共 N 页」取较小值）
python jd_search_crawl.py -n 5 -o sample.csv

# 调整每页滚动：步数 / 步长 / 停顿（网络慢时可略加大停顿）
python jd_search_crawl.py --scroll-steps 15 --scroll-delta 500 --scroll-pause 1.2

# 约 500 条即停（去重后的写入行数；可能在整页处理完后略超一点）
python jd_search_crawl.py -k "内存自营" -o ram.csv -n 80 --max-rows 500
```

### 命令行参数一览（单类 `jd_search_crawl.py`）

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-k` / `--keyword` | 搜索关键词 | `cpu自营` |
| `-o` / `--output` | 输出 CSV 路径 | `jd_search_result.csv` |
| `-n` / `--max-pages` | 最多翻页数 | `100` |
| `--max-rows` | 最多写入多少条商品行（去重后），达到即停 | 不限制 |
| `--listen-mask` | 监听的 URL 子串 | `search-pc-java` |
| `--scroll-steps` | 每页下滑次数 | `12` |
| `--scroll-delta` | 每次下滑像素 | `550` |
| `--scroll-pause` | 每次下滑后等待（秒） | `1.0` |

## 九类自营批量方案（测试用小数据集）

**目标**：CPU、主板、显卡、内存、硬盘、电源、散热、机箱、显示器 共 9 类，每类关键词为 **「品类+自营」**（与 CPU 已成功的方式一致），每类约 **500 条** 去重商品，**9 个 CSV** 分开存放。

**实现思路**：

1. **关键词**：`CPU自营`、`主板自营` … `显示器自营`，已在 `jd_category_jobs.py` 中与文件名绑定；需要改词或加类只改该文件即可。
2. **停表条件**：`--target-rows 500` + `--max-pages` 上限（默认 80）；先到先停，避免无限翻页。500 条按约 30～50 条/页估算，一般十几页内可凑够；结果不足 500 多为结果页本身不够或风控。
3. **执行方式**：**一个浏览器**顺序跑 9 次搜索，类与类之间默认 **暂停 8 秒**（`--pause-between`），减轻触发风控。
4. **存储**：当前阶段用 **9 个 CSV** 做测试库足够；以后要「小数据库」可把 CSV `import` 进 SQLite（同字段建一张 `products` 表 + `category` 列），再按需扩展。

**一键跑九类**（输出到目录 `jd_category_csv`）：

```bash
cd d:\code\claude_code\self_support_crawl
python jd_multi_category_crawl.py --out-dir ./jd_category_csv --target-rows 500
```

**只跑某一类**（文件名或关键词里含「显卡」的条目）：

```bash
python jd_multi_category_crawl.py --out-dir ./jd_category_csv --target-rows 500 --only 显卡
```

**从某一类往后接着跑**（例如前面已跑完 CPU～内存，从硬盘开始到显示器）：

```bash
python jd_multi_category_crawl.py --out-dir ./jd_category_csv --target-rows 500 --from 硬盘
```

`--from` 会匹配 `jd_category_jobs.py` 里**第一个**文件名或关键词包含该字的任务，从该项起向后执行；可与 `--only` 同时使用（先截取再过滤）。

**批量脚本参数**：

| 参数 | 说明 | 默认 |
|------|------|------|
| `--out-dir` | CSV 目录 | `./jd_category_csv` |
| `--target-rows` | 每类目标行数 | `500` |
| `-n` / `--max-pages` | 每类最多翻页 | `80` |
| `--pause-between` | 类与类之间间隔（秒） | `8` |
| `--only` | 只跑匹配该子串的任务 | 空（全跑） |
| `--from` | 从第一个匹配该子串的任务起跑到列表末尾 | 空（从第一项起） |

## 输出 CSV 字段

表头为：商品名称、价格、店铺名称、颜色、评论数、图片、店铺ID、卖点、skuId、总销量、商品ID。  
编码为 **UTF-8 BOM**（`utf-8-sig`），便于 Excel 直接打开。  
跨页、同页多包会按 **skuId**（无则用 wareId）去重后再写入。

## 分页与选择器说明

- **总页数**：匹配 `div[class*="_pagination_total"]` 内「共 N 页」文案。
- **下一页**：匹配 `div[class*="_pagination_next"]`。  
  京东使用 CSS Modules，类名中间的 hash 可能随前端发版变化；若翻页失败，请在开发者工具中重新核对这两个区域是否仍包含 `_pagination_total` / `_pagination_next` 片段，必要时改 `jd_search_crawl.py` 里对应选择器。

## 常见问题

1. **BrowserConnectError / 浏览器连不上**  
   确认 DrissionPage 配置的浏览器路径为 **Chrome 或 Edge**，不要用 Firefox。

2. **listen 收不到包**  
   适当放宽 `--listen-mask`；检查是否在登录/验证码页；网络较慢时增大 `--scroll-pause` 或首包超时（需改源码中 `wait(timeout=25)`）。

3. **总页数解析为 None**  
   分页 DOM 变更会导致解析失败，此时脚本仍按 `--max-pages` 尝试翻页，直到点「下一页」失败为止。
